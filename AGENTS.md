# AGENTS.md

## Critical constraint

**No artefact-specific logic.** This project is a generic test observation platform. Do not add `if family == "snap": ...` branching, family-specific API integrations, or family-conditioned queries. Existing family-specific routes (`/snaps`, `/debs`, `/charms`, `/images`) and external integrations (Snapcraft, Launchpad) are acknowledged tech debt — do not add more.

## Backend (Python/FastAPI)

### Commands (run from `backend/`)

```bash
uv sync --all-groups                    # install all deps (dev + test); plain `uv sync` omits pytest/mypy/ruff
UV_DYNAMIC_VERSIONING_BYPASS=0.0.0 uv sync --all-groups  # required when running outside a built wheel
uv run ruff check . && uv run ruff format --check . && uv run mypy .   # lint → format check → typecheck
uv run pytest                           # run tests (needs running PostgreSQL)
DB_URL="postgresql+pg8000://..." uv run pytest   # custom DB URL
```

### Testing gotchas

- Tests **require a running PostgreSQL**. `conftest.py` auto-detects the host: tries `localhost:5432` first, then `test-observer-db` (Docker). Set `TEST_DB_URL` to override.
- A throwaway test database is created/dropped per session via `sqlalchemy_utils`.
- Use the `generator` fixture (`DataGenerator`) for test data — see `tests/data_generator.py`.
- Use `make_authenticated_request()` or `override_permissions()` context manager for auth-dependent endpoints.

### Key facts

- DB driver is **pg8000** (not psycopg2). All connection strings use `postgresql+pg8000://`.
- FastAPI app has `redirect_slashes=False` — API paths must be exact (no trailing slash).
- Version is dynamic via `uv-dynamic-versioning` (git tags). Set `UV_DYNAMIC_VERSIONING_BYPASS=0.0.0` for local runs.
- Celery + Redis for background tasks. Broker URL: `CELERY_BROKER_URL` env var. Periodic tasks disabled with `DISABLE_PERIODIC_TASKS=true`.
- Alembic migrations run automatically on Docker startup (`dev_entrypoint.sh`). To create: `uv run alembic revision --autogenerate -m "..."`. Always review autogenerated migrations — they are often incomplete (e.g. missing server defaults for new NOT NULL columns).
- Copyright headers required on all source files. REUSE tool is used in CI — see `REUSE.toml` for exempt paths.

## Frontend (Flutter/Dart)

### Commands (run from `frontend/`)

```bash
flutter pub get                                         # install deps
dart run build_runner build --delete-conflicting-outputs  # codegen (REQUIRED after model changes)
flutter analyze                                         # static analysis (run before tests)
flutter test --platform chrome                          # unit tests (requires Chrome)
```

### Codegen

- Models use `freezed` + `json_serializable`; providers use `riverpod_generator`.
- After editing any model or provider, run `dart run build_runner build`. Generated `.g.dart` and `.freezed.dart` files are **committed** to version control.
- The analyzer excludes `**/*.g.dart` and `**/*.freezed.dart` (in `analysis_options.yaml`).

### Integration tests

```bash
chromedriver --port=4444   # must be running first
flutter drive --driver=test_driver/integration_test.dart --target=integration_test/<file> -d chrome
./run_integration_tests.sh  # all integration tests
```

### Runtime config

- `frontend/assets/config.yaml` controls visible tabs and `require_authentication`.
- Default backend URL: `http://localhost:30000/`. Override via `window.testObserverAPIBaseURI` in `web/index.html`.
- Flutter version pinned to **3.29.3**.

## Docker Compose

```bash
docker compose up                          # full stack (backend:30000, frontend:30001, db:5432, saml-idp:8080)
SEED_DATA=false docker compose up          # skip test data seeding
docker compose exec test-observer-api pytest  # run backend tests in container
```

- Backend container uses `network_mode: host` (not port mapping) — required for SAML IdP redirect flow.
- Migrations + seeding run automatically via `dev_entrypoint.sh`.
- SAML IdP test users: `certbot`/`password`, `mark`/`password` (see `backend/saml/simplesamlphp/authsources.php`).

## justfile (optional alternative to Docker Compose)

A `justfile` at repo root provides recipes for running services directly on the host without Docker: `just install-deps`, `just setup`, `just dev`, `just test-backend`, `just lint-backend`, `just test-frontend`, `just lint-frontend`. Prefer `docker compose` for day-to-day development.

## API structure

All endpoints under `/v1/` prefix. Controllers organized by domain in `backend/test_observer/controllers/`. Router at `controllers/router.py`. Health endpoints at `/health/live` and `/health/ready` (no `/v1/` prefix).
